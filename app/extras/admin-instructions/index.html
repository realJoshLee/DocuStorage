Overview

Read the following document to learn more about managing your Tasks deployment.

Getting to the admin panel

To get to the admin panel, navigate to the Settings page and scroll all the way to the bottom. Next, click on Admin Dashboard/Management. Clicking this will open a new page.

When you first open the admin dashboard, you will see a bunch of sections with different stats and useful links to managing the deployment.



Setup Pt.1

To setup this deployment, follow these steps.

1. Move all of files from the GitHub to your webserver. Everything must have it's own website (sub-domain websites are fine as well). This app needs to be in the root of the website.

2. Run the 'database-configure.sql' file located in the 'app/init/' folder.

3. Configure database connection and encryption in the 'config.php' file in the 'app/init/' folder.

Setup Pt.2 (Web interface)

To setup this deployment, follow these steps.

1. Navigate to the url of ur hosted instance.

2. Log in with the default admin account with the following credentials: Username: admin@example.com Password: iX6qAMXk

3. Navigate to the settings and click on the admin button at the bottom of the settings page.

4. You will be directed to the main dashboard of the app. On the top banner that asks you to configure the app, click on HERE and configure all of the settings.

5. From there your instance is ready to use. You can create an account from the home page and promot it to an admin from the admin panel with the default account.

** Make sure that you suspend the default admin once you're done configuring everything. **
You can also view more instructions froom the links in the admin panel.



User management

To manage all of the users who have signed up, in the Actions panel, click on Users. The user management page will load where you can select a user to manage.


Manually adding users

In the add user form, you can add a user. The default password for all users is Tasks1234! if you don't update it. When you add a user through this form, the whitelist feature will not take effect if you have it enabled and configured.


Managing user account (applying permissions)

To manage a certain user, look for their name and email and click on it. You will have some actions and features that you can view. From this panel, you can export all of their data (If enabled), set their preminum status, give them access to the admin panel, mark the user as active or suspend them, or verify them. Clicking the green buttons will put that feature/setting into effect and the red buttons will revoke that permission/settings. In this page you can also view the users full name, email, date the account was created, account id, their timezone, and their theme. If a user is suspeneded, they will not be able to log into their account, but it won't delete any of their data.



App management

To manage all of the app settings, in the Actions panel, click on App Settings. The app management page will load and you will see a large amount of settings.

To enable or disable a feature, click on the dropdown and select Enabled/Disabled then scroll down and select the Update Content button.

Signup Whitelist - When enabled, this will enforce a domain whitelist for users who want to create an account. Domains can be configured in the "app/init/config.php" document under the "$whitelist" variable. Add root domains: example.com of the emails accounts, each domain should be comma seperated.

Magic Link Login - When enabled, users will have the option to login to their account with a link sent to their email account. Mailjet must be setup and configured for this setting to work.

Email Verification - When enabled, users will be required to verify their email address. If diabled, their account with be auto verified on login. Mailjet must be setup and configured for this setting to work.

Log Page - When enabled, the log page will show up in the side nav for all users to view.

Share Page - When enabled, users will be able to share tasks with other people who do and don't have an account. If this is disabled, the share settings will still show in the task but the view page will not work.

Account Creations - When enabled, people can make new accounts. When disable, accounts can't be created unless your in the admin portal.

Bug Reporting - When enabled, users will be able to submit bugs and issues in the settings page.

Allow theme change - When enabled, users will have the option to change their theme.

All data exporting, Login data exporting, App load data exporting - When enabled, the export buttons will show in the user management page. When disabled, these buttons will not appear.

Error reporting - When enabled, system errors (PHP) will show for all users.

Reminder Email - When enabled, reminder emails will be sent from the server. Mailjet must be setup and configured for this setting to work.

IP Whitelist - When enabled, an IP whitelist will be applied. To add IP addresses, add comma seperated IP addresses to the IP Whitelist Value field below. Make sure you have your IP Whitelisted before saving. If you don't have your IP whitelisted you can't access the admin panel and will have to login to the database and disable the feature/add your IP whitelist.

Contact Email - This email will be tied to all of the help links around the app.

Root Website - This is the root url of the deployment (tasks.example.com) and is used around the app.

Email sending control - These are the settings to setup and configure Mailjet. Mailjet is a free resource for sending emails. You must have a domain for this to work. View the Mailjet setup here. Follow the instuctions and select PHP for the app when it asks. Set the values below with values provided by and configured in Mailjet.



Revert to defaults

To revert all of the settings back to their default, click on the dropdown and select "Clear defaults" then click "Revert to install defaults".



Purge logs

This will purge all of the logs from the "tasks_log" database table. These logs can't be retrived unless you have database backups in place. To purge logs click on the drop and select, "Clear log" and then click the "Clear log" button.



Instructions

How to configure whitelist

1. Open the config.php file located in app/init/.

2. Edit the '$whitelist' value and add the root domains that you would like to whitelist.

3. Navigate to the admin panel and then to App Settings.

4. Enable the Signup Whitelist option.
